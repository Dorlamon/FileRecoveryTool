WHAT'S NEW

Current release: v5.8.5.12

(en-US)
-------------------------------------------------------------------------------------------
New in Version 5.8.5.12:

Revised Logic Specifications
Default Behavior: Changed the default setting to skip password-protected files.

Password Input: When the "P" key is pressed, prompt the user to enter a password. This password will then be automatically applied to all encrypted files in the batch to attempt decryption.

Failed Decryption: If the provided password fails to unlock a file, the file should be flagged/marked as usual.

Empty Input: If the "P" key is pressed but no password is entered, the system will continue to skip password-protected files.

Clearing Passwords: Pressing the "P" key again while a password is saved will clear the current password.

-------------------------------------------------------------------------------------------
New in Version 5.8.5.11:

Fix: Corrected the logic so that when [P] Skip encrypted files is OFF, probe errors no longer force encrypted legacy Office files to be skipped.
Behavior: ProbeError now follows the same skip decision as the P toggle.

-------------------------------------------------------------------------------------------
New in Version 5.8.5.10:

Fix: When scanning encrypted legacy Word/Office files with interactive password prompts enabled, the scan now runs the conversion in a helper process to avoid blocking the main scan loop.
Cancel: While waiting for the helper conversion, ESC can now cancel the current helper task.
Behavior: This improves recovery when password prompts appear but the conversion becomes unresponsive.

-------------------------------------------------------------------------------------------
New in Version 5.8.5.9:

Menu: Added a new [P] option to toggle "Skip encrypted files".
Default: The option is ON by default.
Behavior: When turned OFF, legacy Office conversion no longer skips encrypted files before conversion and can fall back to interactive password prompts.
UI: The current setting is shown in the right-side settings panel.

-------------------------------------------------------------------------------------------
New in Version 5.8.5.8:

Encrypted file support: Added detection for encrypted .pdf and .rtf formats.
Visual upgrade: Scanning progress bar upgraded to PRO style.
Visual upgrade: Actual renaming progress bar upgraded to PRO style.
Pre-processing: Display "Calculating files, please wait..." before renaming begins.
Progress Tracking: Full progress bar for the actual renaming process.
Interruption: Press ESC to abort and trigger a UI confirmation prompt.
Reporting: Export an "Aborted Task Report" if the process is cancelled.
Visuals: Generate a color-coded HTML report for actual renaming results.
Integration: Dashboard PRO integrated into the main menu.

-------------------------------------------------------------------------------------------
New Rules in Version 5.8.5.3:

Scope: Applies only to .xls and .xlsx files.
Detection Logic: If the content contains "Roster" characteristic fields, such as:

Name
Date
ID No / ID Number / ID Card Number
Naming Priority: If at least two categories of these characteristics are detected, the suggested filename (in both Simulation and Actual modes) will be prioritized as: Roster.
Date Integration: If a recognizable date is found within the content, it will follow the format: Roster_2025-04-13.
Improvements in this version:
.xlsx handling: Logic now evaluates both the Sheet Name and the Sheet Content.
.xls handling: Logic evaluates the PreviewText extracted during the initial scan.
Confidence Score: I have increased the naming confidence score for this detection type, ensuring the report clearly identifies these as high-confidence results.

v5.8.5.3 這版新增的規則是：

只對 .xls / .xlsx 生效
如果內容中偵測到名冊特徵欄位，像是：
姓名
日期
身份證號 / 身分證號 / 身分證字號
ID No / ID Number
而且至少命中 2 類以上欄位特徵
就會在 模擬改檔名 與 實際改檔名 的建議命名中，優先判定為：
名冊

如果內容裡還有可辨識日期，會變成：

名冊_2025-04-13

這版也一起處理了兩種情況：

.xlsx
會從工作表名稱 + 工作表文字內容做判斷
.xls
會從你原本掃描抽出的 PreviewText 做判斷

另外我把這種判定的命名信心分數也拉高了，所以報表裡會比較明確地顯示這是高可信度命名。
